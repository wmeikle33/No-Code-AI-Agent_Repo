# Framework Layer vs Greenscale Layer

This repository has two complementary purposes:

1. Provide a reusable no-code AI agent framework.
2. Demonstrate that framework through the Greenscale reference implementation.

## Framework Layer

The framework layer contains reusable patterns that can be adapted to different organizations or domains.

Examples include:

- Agent templates
- Workflow templates
- Tool-use patterns
- Routing logic
- Evaluation methods
- Governance policies
- Human-in-the-loop checkpoints
- Monitoring and logging patterns

## Greenscale Layer

The Greenscale layer shows how the framework can be applied to a realistic organization.

Examples include:

- Research monitoring
- Competitor tracking
- KPI monitoring
- Executive brief generation
- Market research
- Report generation
- Document Q&A

## Why This Separation Matters

Separating the framework from the Greenscale implementation makes the project easier to understand, extend, and evaluate.

A reviewer can see both:

- the reusable system design
- the practical business application
